# Uebung 24: Kubernetes Best Practices

Ziel: Ein Deployment mit einer robusteren Variante vergleichen.

## Dateien

- `deployment.bad.yaml`: minimale, aber schwache Variante
- `deployment.good.yaml`: Variante mit Probes, Ressourcen und Replikas

## Aufgaben

1. Vergleiche beide Deployments.
2. Benenne Unterschiede bei Stabilitaet und Betrieb.
3. Erklaere, wofuer `readinessProbe`, `resources` und `replicas` stehen.
