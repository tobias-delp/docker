# Uebung 23: Compose Best Practices

Ziel: Ein unguenstiges Compose Setup mit einer verbesserten Variante vergleichen.

## Dateien

- `compose.bad.yaml`: einfache, aber problematische Variante
- `compose.good.yaml`: strukturiertere Variante mit Healthcheck
- `.env.example`: Beispiel fuer ausgelagerte Variablen

## Aufgaben

1. Vergleiche beide Compose-Dateien.
2. Benenne Verbesserungen bei Struktur, Variablen und Healthcheck.
3. Erklaere, warum `depends_on` kein vollstaendiger Bereitschaftscheck ist.
